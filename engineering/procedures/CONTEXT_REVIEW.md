# CONTEXT REVIEW

## Objetivo

Analizar el estado actual del proyecto antes de implementar una iteración.

El propósito es comprender el contexto existente y evitar asumir cómo está construido el sistema.

---

## Entradas

- SPEC
- Iteración

---

## Actividades

Analizar únicamente los elementos relevantes para la iteración:

- Core
- Configuración
- Módulos involucrados
- Shared
- Middleware
- Tipos
- Utilidades
- Base de datos
- Dependencias

No mostrar el contenido completo de los archivos.

---

## Resultado esperado

Generar un reporte técnico con el siguiente formato.

### Resumen

Estado actual del módulo.

---

### Archivos relevantes

Listado de archivos que participarán en la implementación.

---

### Dependencias existentes

Clases, funciones, utilidades y tipos reutilizables.

---

### Riesgos

Dependencias o restricciones técnicas.

---

### Restricciones para la implementación

Reglas que CODEX deberá respetar.

---

## Restricciones

No escribir código.

No proponer nuevas funcionalidades.

No realizar refactorizaciones.

No modificar la arquitectura.
