# CODE REVIEW

## Objetivo

Validar una implementación antes de cerrar una iteración.

La revisión evalúa únicamente la iteración implementada.

No evalúa el SPEC completo.

---

## Entradas

- SPEC
- Iteración

---

## Verificar

1. Alcance de la iteración.

2. Arquitectura.

3. Responsabilidades.

4. Dependencias.

5. DEC.

6. ADR.

7. Riesgos.

8. Criterios de aceptación.

---

## Resultado esperado

Responder únicamente:

✅ Aprobado

o

❌ Requiere cambios

Explicando brevemente cada punto.

---

## Restricciones

No proponer nuevas funcionalidades.

No refactorizar.

No ampliar el alcance.

No evaluar iteraciones futuras.

La revisión debe limitarse a validar el cumplimiento de la iteración correspondiente.
